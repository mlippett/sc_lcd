Character display Application¶

The character display application shows how to display character and text on the LCD module. The application is intended to use the LCD component as a standalone component, which means there is no image buffer to store the LCD frame data. The application writes directly to the LCD component.

Features supported in the application¶

The application supports the following:

  1. Setting the graphics frame ready for display
  2. Setting the background colour
  3. Setting the foreground colour
  4. Reading the background colour
  5. Reading the foreground colour
  6. Setting the space between characters in a text
  7. Setting the text style - can be none, Italics, bold, bold and Italics
  8. Displaying a character of size 6 * 6 pixel in monochrome colour
  9. Displaying a character of size 6 * 6 pixel in colour
  10. Displaying text of size 6 * 6 pixel in monochrome colour
  11. Displaying text of size 6 * 6 pixel in colour (mentioned foreground colour is used)
  12. Displaying text of size 16 * 16 pixel in colour (mentioned foreground colour is used)
  13. Putting an image to the screen

Project structure¶

The application includes character specific functionalities and general graphics functionalities. The general graphics functionalities are available in the files

  • graphics.xc
  • graphics.h

The character specific functionalities are available in the files

Project structure
File Description
character_display.h Header file containing the APIs for the character and text writes
character_display.xc File containing the implementation of the character and text write for 6 * 6 pixel and 16 * 16 pixel
alphabet.c File containing the pixel data for 6 * 6 characters. Currently the file includes only A-Z (upper case) and space
alphabet_16x16.c File containing the pixel data for 16 * 16 characters. Currently the file includes only A-Z (upper case) and space. It also includes the font with no formatting and in Italics
alphabet_lookup_table.c File containing the lookup table for the characters and APIs to find the pixel data for a character
character_specific_includes.h Header file including the defines specific for the character list

APIs¶

Following are the list of APIs that can used for character and text display. Currently only 6 * 6 and 16 * 16 pixel fonts are supported

void set_background_color(chanend client, unsigned short color)¶

The function sets te background color for the LCD screen.

Parameters:
  • client – The LCD channel number
  • color – The color to be set for the background color

At start-up the graphics initialization sets the color to black

void set_foreground_color(unsigned short color)¶

The function sets the foreground color in the LCD screen.

Parameters:
  • client – The LCD channel number
  • color – The color to be set for the foreground color

At start-up the graphics initialization sets the color to white

The text written on the screen takes the foreground color

void set_graphics_frame(chanend client)¶

The function initializes the graphics screen - includes setting the background and foreground color.

Parameters:
  • client – The LCD channel number
unsigned short get_foreground_color(void)¶

The function reads the foreground colour set.

Returns:The foreground colour set
unsigned short get_background_color(void)¶

The function reads the background colour set.

Returns:The background colour set
void put_image_BW(chanend client, unsigned start_row, unsigned start_col, unsigned height, unsigned width)¶

The display an image on the screen.

The image will be displayed in monochrome colour

Parameters:
  • client – The LCD channel number
  • start_row – The row to start the image
  • start_col – The column to start the image. Should be in multiples of 2 as the LCD write is done in words and not in pixels
  • height – Height of the image in rows
  • width – Width of the image in pixels

The 0s in the image structure is displayed in BLACK and the 1s in the image structure is displayed in WHITE

void write_text_6x6_bw(chanend client, unsigned start_row, unsigned start_col, char text_to_type[])¶

To write a text with fonts of pixel size 6 * 6.

Parameters:
  • client – The LCD channel number
  • start_row – Start row for the text
  • start_col – Start column for the text
  • text_to_type[] – The array holding the text to type

Each text string should end with indicating the end of text

The user should make sure the text size doesn’t exceed the LCD width size. Else the code will have unexpected behaviour

The user should make sure the character size also doesn’t exceed the LCD limits

The API displays the text in monochrome colour

The current code supports only A- Z (upper case) and “space”

Demo¶

The file demo.xc includes a sample demo using the character and text display and displaying the image. The function c:func::graphics_demo does the following

  • Displays the image of a silicon on the screen. The Silicon image keeps moving on the screen

  • The image is displayed as monochrome image

  • Multiple text that keeps changing after a certain time of display

  • The text is displayed in Yellow colour and in Italics

  • The text displayed are
    • WELCOME TO XMOS
    • HIGH SPEED USB
    • MOTOR CONTROL
    • ETHERNET AVB
    • FREE TOOLS

Executing the application¶

The application is available as app_graphics_demo. The following steps ahould be followed to run the application

  1. Include the app_graphics_demo and module_lcd to the workspace
  2. Build the application
  3. Run the application using the XTAG
  4. The image of the silicon and the texts keeps repeating on the screen
